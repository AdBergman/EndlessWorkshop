## Scope

Implemented `CDEX-EXP-004` only, as a conservative safe subset.

Changed:

- Existing cached `ActionCostModifierDefinition.ActionTypes` relationships now project into `Action mechanics` instead of a separate `Cost modifiers` product section.
- Existing cached/decompile-backed action formula projections continue to project into `Action mechanics`.
- Cost modifier and formula items are merged into one `Action mechanics` section per action.
- Public action mechanics wording no longer mentions DB/RPN/source provenance.
- Formula reference keys are promoted into `referenceKeys` only when the key is a known public Codex prefix.
- Action validation now fails if technical action wording such as DB/RPN, descriptor/tag/class/provenance/debug terms leaks into public action text.

Not changed:

- No new action source probing.
- No UI mapper probing.
- No broad fallback probing.
- No invented action purpose summaries.
- No Resource, Trait, Quest, or broad thin entity cleanup.
- No Codex root shape, `exportKind`, filename, or import path changes.

## Contract Compatibility

No EWShop importer migration should be required.

The pass uses existing generic Codex fields only:

- `sections`
- `sections[].items[]`
- `sections[].items[].referenceKey`
- `sections[].items[].facts[]`
- `referenceKeys`
- `publicContextKeys` generated by existing Codex metadata rules

## Source Rules

Action metadata is emitted only from already-cached exporter sources:

- `ActionCostModifierDefinition.ActionTypes`
- decompile-backed static action formula key relationships already present in the exporter
- `EmpireActionDefinition` and `ConstructibleActionDefinition` formula/cost fields already used by the cached formula resolver

Unresolved or ambiguous actions remain facts-only/link-only.
